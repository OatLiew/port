CNN Crawler

CNN Crawler includes four majors parts, connecting to EC2 for JSONP callback for the NBA players stats which are on AWS RDS, buiding trie data structure for search suggestion with WIKI pedia titles on Webservice with C#, and crawling through CNN website to index titles with wokerroles and returning the search result with Azure tables(no sql) which is displayed on the dashboard. The site is hosted on Azure and Azure table, queues AWS EC2 and RDS.



Technology: ASP.net, C#, JSON, JSONP, WEBServices, AJAX, Jquery, Bootstrap, Javascript, html, CSS, AWS RDS, AZURE, Azure table

The users would be able to search for NBA players name and be able to see the display pictures and their stats. There would be a suggested query from WIKIpedia titles from a-c (running out fo memory) and the users would be able to search for any crawled words title in the CNN.com website. The result will include title, date and urls linked.

For the first part, the NBA player stats are on AWS RDS. I created a php webservices with a callback funtion for JSONP. The returning result would be the player Stats with display pictures. The name need to be an exact match for the stats to show up.

For the second part, I built a static trie data structure from WIKI pedia tiles, the trie is able to build up to "C" because of the running out of memory issue. The code is written on C# Webrole. Also, I use keep alive service to keep the trie data active.

For the third part, the worker role will first crawl through robots.txt on CNN to get all the urls then put into Azure queues. Then the woker will start crawling until the stop command is issued. The web role and worker role can commnicate through the Azure queues. It will index the page title and date on each links in CNN.com site and upload the key pair value to Azure table(no sql) for retrieval of the title and urls.

The last part is just to build up the dashboard for displaying the information and issuing command to the worker role.
